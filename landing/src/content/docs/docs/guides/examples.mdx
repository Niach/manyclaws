These examples show typical interactions between friends and ManyClaws agents. The agent handles Kubernetes orchestration behind the scenes while friends interact through natural conversation.

## Vibe-Coding: Friend asks agent to build something

```
Friend (alice): Hey, can you build me a simple todo app?

Agent: Klar! I'll spin up a todo app for you. Let me check your namespace first...

Agent: [runs: GET /api/friends/alice/namespace/status]
       Your namespace is healthy — 0/3 pods used, plenty of room.
       I'll deploy a Node.js todo app with persistent storage.

Agent: [runs: POST /api/friends/alice/namespace/apply]
       [deploys Deployment + Service for todo-app]

Agent: App is running! Let me expose it so you can reach it from the internet.

Agent: [runs: POST /api/friends/alice/namespace/expose]
       {"name": "todo", "service": "todo-app", "port": 3000}

Agent: Done! Your todo app is live at https://todo-alice.your-domain.net
       It's a simple web UI — just open the link and start adding tasks.
```

<Aside type="tip">
The agent deploys to the friend's isolated `friend-alice` namespace, not its own. Resource quotas and network policies protect the cluster.
</Aside>

## Portal Access: Friend manages secrets

```
Friend (bob): I need to store my OpenAI API key somewhere safe.
              Can your apps use it?

Agent: I'll send you a portal login link — you can manage secrets
       securely from there. One moment...

Agent: [runs: POST /api/auth/portal-session {"friend_id": "bob"}]

Agent: Here's your portal link:
       https://my-agent.your-domain.net/portal?s=abc123...

       Click it to access your personal dashboard. Under the
       "Secrets" tab you can add your OpenAI key. The link is
       valid for 30 days.

       Once you store the key, any app I deploy to your namespace
       can reference it.
```

```
[Bob opens the portal, clicks "Secrets", creates a new secret:]

Secret name: openai-key
Key:         OPENAI_API_KEY
Value:       sk-...
```

<Aside>
Secrets are stored as Kubernetes Secrets in the friend's own namespace. Only workloads in that namespace can access them.
</Aside>

## Adding Infrastructure: Friend extends their app

```
Friend (alice): My todo app needs a database. Can you add Postgres?

Agent: Sure! I'll deploy Postgres to your namespace and wire it up.

Agent: [runs: POST /api/friends/alice/namespace/apply]
       [deploys postgres Deployment + Service + PVC]

Agent: [runs: POST /api/friends/alice/namespace/apply]
       [updates todo-app Deployment with DATABASE_URL env var
        pointing to postgres.friend-alice.svc:5432]

Agent: Postgres is running in your namespace with 256Mi storage.
       I've updated the todo app to connect to it automatically.

       Your app is still live at https://todo-alice.your-domain.net
       Want me to add anything else? Maybe a backup cron job?
```

```
Friend (alice): Yeah, back it up daily at 3am.

Agent: [runs: POST /api/friends/alice/namespace/apply]
       [deploys CronJob: pg_dump at "0 3 * * *"]

Agent: Done! A daily Postgres backup runs at 3:00 AM.
       The dump is stored in a PVC in your namespace.
```

```
Friend (alice): What's running in my namespace right now?

Agent: [runs: GET /api/friends/alice/namespace/status]

Agent: Here's your namespace overview:

       Pods (3/3 quota):
       - todo-app-7d8f9b-x4k2m    Running  (128Mi)
       - postgres-5c6d7e-n3p1q    Running  (192Mi)
       - pg-backup-28473892-abc    Completed

       Services:
       - todo-app    ClusterIP  :3000
       - postgres    ClusterIP  :5432

       Exposed:
       - https://todo-alice.your-domain.net → todo-app:3000

       Quota: 300m/500m CPU, 384Mi/512Mi RAM

       Everything looks healthy!
```

## Cron Jobs: Friend sets up scheduled tasks

```
Friend (bob): Can you remind me every morning at 9am to check
              my portfolio?

Agent: I'll set up a cron job for that.

[Agent creates a cron via the workspace cron system]

Agent: Done! Every day at 9:00 AM (Europe/Berlin), I'll send you
       a message to check your portfolio. You can manage this in
       your portal under "Cron Jobs" — pause, edit, or delete it
       anytime.
```

## Multi-Channel: Same friend, different platforms

```
[Signal message]
Friend (alice): Hey, what's the status of my todo app?

Agent: Still running great! 3 pods healthy, all services up.
       https://todo-alice.your-domain.net is live.

[Later, on Discord]
Friend (alice): Can you add a dark mode toggle to the todo app?

Agent: Sure! I remember you have the todo app running in your
       namespace. Let me update the deployment with a new image
       version that includes dark mode...
```

<Aside>
The agent recognizes Alice across Signal and Discord through the cross-platform identity system. Each channel gets its own session, but the agent has access to the same friend workspace files.
</Aside>
