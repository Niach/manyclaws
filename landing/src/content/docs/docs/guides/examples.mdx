These examples show typical interactions between friends and ManyClaws agents. The agent handles Kubernetes orchestration behind the scenes while friends interact through natural conversation.

## Vibe-Coding: Friend asks agent to build something

<ChatWindow title="Signal DM — alice">
  <ChatMessage from="friend" name="Alice">
    Hey, can you build me a simple todo app?
  </ChatMessage>

  <ChatMessage from="agent" name="Agent">
    Klar! I'll spin up a todo app for you. Let me check your namespace first...
  </ChatMessage>

  <ChatAction action="GET /api/friends/alice/namespace/status" />

  <ChatMessage from="agent" name="Agent">
    Your namespace is healthy — 0/3 pods used, plenty of room. I'll deploy a Node.js todo app with persistent storage.
  </ChatMessage>

  <ChatAction action="POST /api/friends/alice/namespace/apply → Deployment + Service" />

  <ChatMessage from="agent" name="Agent">
    App is running! Let me expose it so you can reach it from the internet.
  </ChatMessage>

  <ChatAction action="POST /api/friends/alice/namespace/expose → todo-alice.your-domain.net" />

  <ChatMessage from="agent" name="Agent">
    Done! Your todo app is live at `https://todo-alice.your-domain.net` — just open the link and start adding tasks.
  </ChatMessage>
</ChatWindow>

<Aside type="tip">
The agent deploys to the friend's isolated `friend-alice` namespace, not its own. Resource quotas and network policies protect the cluster.
</Aside>

## Portal Access: Friend manages secrets

<ChatWindow title="Discord DM — bob">
  <ChatMessage from="friend" name="Bob">
    I need to store my OpenAI API key somewhere safe. Can your apps use it?
  </ChatMessage>

  <ChatMessage from="agent" name="Agent">
    I'll send you a portal login link — you can manage secrets securely from there. One moment...
  </ChatMessage>

  <ChatAction action="POST /api/auth/portal-session → creates 30-day session" />

  <ChatMessage from="agent" name="Agent">
    <p>Here's your portal link:</p>
    <p><code>https://my-agent.your-domain.net/portal?s=abc123...</code></p>
    <p>Click it to access your personal dashboard. Under the "Secrets" tab you can add your OpenAI key. The link is valid for 30 days.</p>
  </ChatMessage>
</ChatWindow>

<ChatWindow title="Friend Portal — bob">
  <ChatAction action="Bob opens portal → Secrets tab → New Secret" />

  <ChatMessage from="friend" name="Bob">
    <p><strong>Secret name:</strong> <code>openai-key</code></p>
    <p><strong>Key:</strong> <code>OPENAI_API_KEY</code></p>
    <p><strong>Value:</strong> <code>sk-...</code></p>
  </ChatMessage>

  <ChatAction action="Secret created in friend-bob namespace" />
</ChatWindow>

<Aside>
Secrets are stored as Kubernetes Secrets in the friend's own namespace. Only workloads in that namespace can access them.
</Aside>

## Adding Infrastructure: Friend extends their app

<ChatWindow title="Signal DM — alice">
  <ChatMessage from="friend" name="Alice">
    My todo app needs a database. Can you add Postgres?
  </ChatMessage>

  <ChatMessage from="agent" name="Agent">
    Sure! I'll deploy Postgres to your namespace and wire it up.
  </ChatMessage>

  <ChatAction action="POST /api/friends/alice/namespace/apply → Postgres Deployment + Service + PVC" />
  <ChatAction action="POST /api/friends/alice/namespace/apply → Update todo-app with DATABASE_URL" />

  <ChatMessage from="agent" name="Agent">
    <p>Postgres is running in your namespace with 256Mi storage. I've updated the todo app to connect to it automatically.</p>
    <p>Your app is still live at `https://todo-alice.your-domain.net` — want me to add anything else? Maybe a backup cron job?</p>
  </ChatMessage>

  <ChatMessage from="friend" name="Alice">
    Yeah, back it up daily at 3am.
  </ChatMessage>

  <ChatAction action="POST /api/friends/alice/namespace/apply → CronJob pg_dump at 0 3 * * *" />

  <ChatMessage from="agent" name="Agent">
    Done! A daily Postgres backup runs at 3:00 AM. The dump is stored in a PVC in your namespace.
  </ChatMessage>
</ChatWindow>

## Cron Jobs: Friend sets up scheduled tasks

<ChatWindow title="Signal DM — bob">
  <ChatMessage from="friend" name="Bob">
    Can you remind me every morning at 9am to check my portfolio?
  </ChatMessage>

  <ChatAction action="Creates workspace cron → 0 9 * * * Europe/Berlin" />

  <ChatMessage from="agent" name="Agent">
    Done! Every day at 9:00 AM (Europe/Berlin), I'll send you a message to check your portfolio. You can manage this in your portal under "Cron Jobs" — pause, edit, or delete it anytime.
  </ChatMessage>
</ChatWindow>

## Multi-Channel: Same friend, different platforms

<ChatWindow title="Signal DM — alice">
  <ChatMessage from="friend" name="Alice" channel="Signal">
    Hey, what's the status of my todo app?
  </ChatMessage>

  <ChatMessage from="agent" name="Agent">
    Still running great! 3 pods healthy, all services up. `https://todo-alice.your-domain.net` is live.
  </ChatMessage>
</ChatWindow>

<ChatWindow title="Discord DM — alice">
  <ChatMessage from="friend" name="Alice" channel="Discord">
    Can you add a dark mode toggle to the todo app?
  </ChatMessage>

  <ChatMessage from="agent" name="Agent">
    Sure! I remember you have the todo app running in your namespace. Let me update the deployment with a new image version that includes dark mode...
  </ChatMessage>

  <ChatAction action="POST /api/friends/alice/namespace/apply → Updated todo-app image" />
</ChatWindow>

<Aside>
The agent recognizes Alice across Signal and Discord through the cross-platform identity system. Each channel gets its own session, but the agent has access to the same friend workspace files.
</Aside>
