Secrets are stored as Kubernetes Secrets in friend namespaces. The portal API provides CRUD operations, and the admin/agent API provides listing.

## Portal Endpoints (Friend Session)

### List Secrets

<ApiEndpoint method="GET" path="/api/portal/secrets" auth="friend" description="List all secrets in the friend's namespace." />

**Response:**

```json
[
  {"name": "openai-key", "type": "Opaque", "keys": ["OPENAI_API_KEY"]},
  {"name": "github-token", "type": "Opaque", "keys": ["GITHUB_TOKEN"]}
]
```

### Get Secret

<ApiEndpoint method="GET" path="/api/portal/secrets/:name" auth="friend" description="Read a specific secret's data (base64-decoded values)." />

**Response:**

```json
{
  "name": "openai-key",
  "data": {
    "OPENAI_API_KEY": "sk-..."
  }
}
```

### Create Secret

<ApiEndpoint method="POST" path="/api/portal/secrets" auth="friend" description="Create a new secret in the friend's namespace." />

**Body:**

```json
{
  "name": "openai-key",
  "data": {
    "OPENAI_API_KEY": "sk-..."
  }
}
```

Returns `409` if the secret already exists.

### Update Secret

<ApiEndpoint method="PUT" path="/api/portal/secrets/:name" auth="friend" description="Replace a secret's data (delete + recreate)." />

**Body:**

```json
{
  "data": {
    "OPENAI_API_KEY": "sk-new-key..."
  }
}
```

### Delete Secret

<ApiEndpoint method="DELETE" path="/api/portal/secrets/:name" auth="friend" description="Delete a secret from the friend's namespace." />

## Admin/Agent Endpoints

### List Friend Secrets

<ApiEndpoint method="GET" path="/api/friends/:id/secrets" auth="admin" description="List secrets in a friend's namespace (admin view)." />
